## Summary

Phase 1A for `vacancy_id -> job_id` migration.

This PR lays the storage/runtime groundwork for the split:

- `job_id` as canonical external identity
- `job_setup_id` as internal recruiter setup/runtime key
- `vacancy_id` kept as compatibility alias

It closes `#142` and contributes to parent `#146`.

## What Changed

- added management migration `005_playbook_session_job_identity.sql`
- extended `management.playbook_sessions` flow to persist and resolve:
  - `job_id`
  - `job_setup_id`
  - legacy `vacancy_id`
- updated playbook runtime session identity propagation to carry `job_setup_id`
- added unit coverage for management store identity resolution
- added issue-plan artifacts for parent/child breakdown

## Notes

- This PR is additive only. It does not remove legacy `vacancy_id` paths yet.
- Frontend/API canonical `job_id` cutover is deferred to `#144`.
- Cleanup and rollout hardening stay in `#145`.
