- management store session lookup and abort logic now use `job_setup_id` as the canonical runtime key instead of `COALESCE(job_setup_id, vacancy_id)`

- added management migration `006_playbook_session_job_setup_runtime.sql`
  - backfills `job_setup_id`
  - removes vacancy-based active indexes
  - adds active unique index on `job_setup_id`
- updated unit coverage for the new management-store contract
