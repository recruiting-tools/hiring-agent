# PR: Child 2 `job_id` contract cutover

Closes #144

## What changed

- made `/api/jobs` and `/api/vacancies` return `job_id` as the canonical external identifier
- added `job_setup_id` to recruiter-facing job list payloads as internal runtime metadata
- moved recruiter UI selector persistence from `vacancy_id` storage to `job_id` storage, while still reading the legacy key during rollout
- switched report links and report rendering to `job_id`-first output
- kept `vacancy_id` available as a compatibility alias in websocket/chat flows
- added integration coverage for:
  - `job_id`-first report access
  - `job_id`-first websocket input without `vacancy_id`
  - `job_setup_id` metadata in recruiter job list responses
